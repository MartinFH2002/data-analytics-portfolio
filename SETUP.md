## Daily Workflow

### Committing Changes

1. Stage changes: `git add .`
2. Commit with message: `git commit -m "phase{n}: description"`
3. Push to GitHub: `git push`

### Commit Message Convention

Format: `phase{n}: clear and specific description`

Examples:
- phase1: completed python loops and conditionals exercises
- phase2: implemented pandas groupby and pivot table practice
- phase3: sql window functions and CTEs study

### Before Starting a Session

Always pull latest changes: `git pull`

## Important Notes

- Commits are documentation - Always write specific messages
- Notes in Markdown - Write personal summaries, not copied docs
- Consistency - Commit daily after completing sections
- GitHub Profile - Green heatmap shows dedication and commitment
